TCP Protocol

TCP (Transmission Control Protocol) is a connection-oriented communication protocol that is used to ensure reliable data transmission between devices over a network. It guarantees that data packets arrive at their destination in the correct order and without errors. TCP is commonly used for applications where reliability is crucial, such as web browsing (HTTP), email (SMTP), and file transfer (FTP).

TCP Protocol Parameters

TCP protocol has several key parameters that govern its behavior:

  • Sequence Number: Identifies the order of data packets.
  • Acknowledgment Number: The number that confirms receipt of data packets.
  • Window Size: The amount of data that can be sent before requiring an acknowledgment.
  • Checksum: Ensures the integrity of data by verifying that the data has not been altered during transmission.
  • Flags: Control bits used for various functions like connection initiation and termination (e.g., SYN, ACK, FIN).

TCP checksum

Checksum is an inverted sum of parts:

  1. Pseudo header: hexadecimal number created from parts in binary sourceIP,destinationIP,protocol, length of header and payload
  2. TCP header: hexadecimal number created from parts in binary, Source port, destination port, sequence number, acknowledgment, flag, window size, checksum (0000), urgent pointer
  3. Payload: content of packet written in hexadecimal

TCP Connection

Three-Way Handshake

The three-way handshake is the process by which a TCP connection is established between a client and server:

  1. Step 1: The client sends a SYN (synchronize) message to the server to initiate the connection with sequence number .
  2. Step 2: The server responds with a SYN-ACK message to acknowledge the request this means it responds with sequence number +1 and it's own sequence.
  3. Step 3: The client sends an ACK message to confirm the server's acknowledgment which is original sequence+1 and servers sequence +1. The connection is now established, and data can be exchanged.

Four-Way Handshake

Four-way handshake is the process allowing for orderly closure of the connection between client and server.

  1. Step 1: The client sends a FIN flagged packet witha a sequence number.
  2. Step 2: The server respondes with a ACK flagged packet with a sequence of FIN packet + 1. Now the connection is half-closed and only the last packet will be sent
  3. Step 3: The server sends its own FIN flagged packet with another sequence number indicating that it is ready to close the connection.
  4. Step 4: The client responds with ACK flagged packet with with a sequence of FIN packet + 1. Now the connection is terminated.

UDP Protocol

UDP (User Datagram Protocol) is a connectionless communication protocol. Unlike TCP, UDP does not establish a connection before sending data and does not guarantee the delivery of packets. It is faster than TCP but less reliable. UDP is commonly used in applications that prioritize speed over reliability, such as video streaming, online gaming, and DNS lookups.

UDP Protocol Parameters

UDP has fewer parameters than TCP because it is connectionless. Key parameters include:

  • Source Port: The sending application's port number.
  • Destination Port: The receiving application's port number.
  • Length: The length of the UDP packet.
  • Checksum: Used to check the integrity of data (optional in IPv4 but mandatory in IPv6).
